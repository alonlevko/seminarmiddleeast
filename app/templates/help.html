In this Page the functionality of this website will be explained

View Twitter Data Page:

The main goal of this page is to display tweets, twitter users and apply analytics on them The page is divided into 3 parts. The first line of controls in the settings part, in this part query parameters are defined by the user. The second line of controls is the actions part, here the user can pick which query to run. The last part, That is all that remains from the page, is the results part. Here the results of the query the user has run are displayed.

Settings Part

The locations that are used in this part are configured in the "Configure Locations" page. The first control is a dropdown that enables selecting multiple regions and called "select Region". When a region is selected, the second dropdown "Select Place", contains all places inside of that region that were configured in the "Configure Locations" page. Then these places can be selected and the user will be able to run queries on the twitter data saved on these places. The next controls are the "Start Date" and "End Date". In this controls the user can define on what dated the data he wants to see was created in twitter. All queries are time bases, so this effects every action the user does. The final control is the search words, here the user can configure what search words to use in his queries using the dropdown list. Also, in the last control the user can add or remove a search word. Search words can also be phrases, and they can contains whitespace. Example search words can be: "Hamburger", "Good Icecream", etc..

Actions and Results Part

In this part, based on the setting the user has picked in the settings part, he can run queries on the data saved by our server. The data will show on the results part. Next, The functionality of every control will be explained and then what the results looks like.

Inspect Tweets:

This button lets the user view tweets that are saved in our database. Tweets are filtered using the settings. Tweets shown are from the locations selected based on twitter search, published in the dates that were picked and Containing One of the search words chosen. Tweets are displayed in a table. The first column of the table is the id of the tweet, and it's also a link to the original tweet on twitter. The second column is the text of the tweet (what the user wrote). The third column is the screen name of the user, with a link to his profile page on twitter. The third column is the location the user wrote in his profile on twitter. The coordinated column is the coordinated in the geo part of the tweet if they are present. If twitter recognized a place that the tweet is connected to, it will appear in the place full name column. Created at is the date and time the tweet was created. the other columns are a paid feature, And they are calculated by ibm watson text understanding.
